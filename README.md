# VibeCode IDE – браузерная IDE с прокторингом

Многостраничный адаптивный сайт на React + TypeScript по макетам из `screens/`. Интерфейс представляет браузерную IDE с встроенным прокторингом для хакатона **VibeCode Jam**: фиксирует вставки, DevTools, подозрительные расширения и готов отдавать события на FastAPI/WebSocket для AI-скоринга.

## Стек
- Vite + React 18
- TypeScript
- React Router v6
- CSS Modules

## Соответствие страниц макетам
- `/` – **Перед собеседованием** (`screens/Before interview.png`)
- `/camera` – **Камера** (`screens/Camera.png`)
- `/microphone` – **Микрофон** (`screens/Microphone.png`)
- `/screen` – **Экран** (`screens/Screen.png`)
- `/interview` – **Интервью** (`screens/Interview.png`)

Модальное подтверждение завершения звонка запускается на странице интервью по кнопке «Завершить звонок» и повторяет макет `screens/Модалка.png`.

### Прокторинг и IDE
- Отслеживаем Ctrl+C/Ctrl+V (особенно вставки >500 символов) и открытие DevTools.
- Детектируем AI-расширения (Copilot, ChatGPT и др.).
- Готовый каркас под FastAPI + WebSocket: события отправляются <50мс и могут анализироваться SciBox (qwen3-coder + bge-m3).
- Риск оценивается 0–100%: >70 — высокий риск. Цветовая индикация заложена в карточках.

## Установка и запуск
```bash
npm install
npm run dev       # локальная разработка
npm run build     # продакшн-сборка
npm run preview   # предпросмотр собранного проекта
```

## Структура
- `src/pages` – по одному файлу на экран
- `src/components` – переиспользуемые блоки (layout, UI, секции)
- `src/styles` – глобальные переменные и базовые стили

Header и footer общие для всех страниц, навигация совпадает с экранами из дизайна. Все блоки адаптивные и перестраиваются на мобильных.
